# Cobot审计报告一键处理工具

> 自动化转换Cobot代码审计原始报告，快速生成标准问题单格式，节省人工处理时间，避免操作失误。

## 🎯 项目背景

在中国软件评测中心的代码审计工作中，使用Cobot工具扫描后，会导出一份包含全部问题的Excel原始报告（源文档）。在实际流程中，需要将这份报告转换为格式统一、供研制方填写的问题单（过程文档）。

传统纯人工处理方式存在两大痛点：

1. **费时费力**：需要手动复制、粘贴、调整多列数据。
2. **容易出错**：在整理大量数据时，易发生错行、漏列等人为失误。

本工具正是为解决这一痛点而生，实现从 **【源文档】** 到 **【问题单】** 的一键式自动化转换。

## ✨ 核心功能

- **一键转换**：运行程序，输入源文件名，即可自动生成标准格式的中间过程文档 (`Cobot过程文档.xlsx`)。
- **智能整理**：自动从源文档的特定列（A, D, H, I, K）提取关键信息，并重组到新表格的对应位置。
- **公式生成**：自动在“漏洞位置”列合成文件路径与行号信息（格式：`文件路径 第X行`）。
- **格式美化**：自动为生成的表格添加加粗灰色表头、设置所有内容居中、自动换行以及统一列宽。
- **便捷查看**：处理完成后自动用系统默认程序打开生成的Excel文件。

## 🚀 快速开始

### 方法一：使用可执行文件 (推荐，无需安装环境)

1. 从 [Releases](你的Github Release页面链接) 页面下载最新的 `Cobot_processExcel.exe`。

2. 将下载的exe文件与Cobot导出的**源Excel文档**（如 `xxx的全部问题数据.xlsx`）放置在同一文件夹下。

3. 双击运行 `Cobot_processExcel.exe`。

   > **注意**：由于是PyInstaller打包，程序启动可能有约15秒延迟，属正常现象。

4. 根据控制台提示，输入源文档的文件全名（如：`15s的全部问题数据.xlsx`），然后按下回车。

5. 程序将自动生成并打开名为 `Cobot过程文档.xlsx`的新文件。

### 方法二：运行Python源代码

**前提**：确保已安装Python 3.x 及 `openpyxl`库。

```
# 安装依赖库
pip install openpyxl

# 运行脚本
python Cobot_processExcel.py
```

之后根据提示输入源文件名即可。

## 🔧 输入与输出

### 输入 (源文档)

程序会从您指定的Excel文件中读取以下列：

- **A列**: 问题编号 (Sequence)
- **D列**: 问题严重等级 (Level)
- **H列**: 文件路径 (File Name)
- **I列**: 行号 (Row Number)
- **K列**: 问题描述 (Detail)

### 输出 (过程文档)

生成的 `Cobot过程文档.xlsx`将包含以下规范列：

| 列   | 标题         | 内容来源/说明                                    |
| ---- | ------------ | ------------------------------------------------ |
| A    | 问题编号     | 直接取自源文件A列                                |
| B    | 问题名称     | *(留空，供后续填写)*                             |
| C    | 问题严重等级 | 直接取自源文件D列                                |
| D    | 文件路径     | 直接取自源文件H列                                |
| E    | 行号         | 直接取自源文件I列                                |
| F    | **漏洞位置** | **自动合成**，公式为 `=D{row}&" 第"&E{row}&"行"` |
| G    | 问题描述     | 直接取自源文件K列                                |
| H    | 是否修改     | *(留空，供研制方填写)*                           |
| I    | 修复措施     | *(留空，供研制方填写)*                           |

## ⚙️ 代码自定义

如果需要调整处理的列、输出格式或文件名，可以修改 `Cobot_processExcel.py`中的相关变量：

- 修改 `ws1['A']`等列表推导式来改变读取的源数据列。
- 修改 `ws2[f‘A{row}’]`等赋值语句来调整输出位置。
- 在 `wb2.save()`中修改输出文件名。

## ❓ 常见问题

- **Q: 运行exe文件时报错“找不到文件”**

  A: 请确保源Excel文件与exe文件在**同一目录**，并且输入的文件名**完全正确**（包括`.xlsx`后缀）。

- **Q: 生成的表格列宽或样式不合适**

  A: 可在源代码中调整 `ws2.column_dimensions[].width`的值修改列宽，或修改 `Font`, `Alignment`, `PatternFill`的参数调整样式。

- **Q: 如何在Mac/Linux上运行？**

  A: 源代码中已注释掉针对Mac和Linux的自动打开文件命令。若在这些系统上使用Python运行，需取消注释对应系统的 `subprocess.Popen`命令。
